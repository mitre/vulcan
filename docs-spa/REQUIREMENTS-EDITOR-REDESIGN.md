# Requirements Editor - Modern UX Redesign

**Created:** 2025-12-02 (Session 82)
**Status:** Design Proposal

---

## Current Problems

1. **Split Context** - Component view vs Controls view, constant switching
2. **Overwhelming** - 250+ rules visible, no workflow guidance
3. **No Progress** - Can't see "how done" the component is
4. **Rigid Layout** - Same UI for triage, authoring, and review

---

## User Workflow Analysis

### The Real Tasks

| Phase | Task | Volume | Time Spent |
|-------|------|--------|------------|
| 1. Triage | Bucket rules into statuses | 250 rules | 2-4 hours |
| 2. Author | Write check/fix for Configurable | ~200 rules | Days/weeks |
| 3. Review | Approve/request changes | ~200 rules | Hours |
| 4. Release | Final check, release component | 1 action | Minutes |

### Key Insight

**Different phases need different UIs.** Triage is bulk categorization. Authoring is deep focus. Review is decision-making.

5. Expands RHEL 8 STIG to see their check/fix text
6. Clicks "Copy" to append content to current rule
7. Modifies copied text for their specific platform

### Industry Patterns

| Tool | Table View | Board View | Key Insight |
